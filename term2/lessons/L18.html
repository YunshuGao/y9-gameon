L18 · Week 9 ⏱ 75 minutes 🎯 IND5-2 · IND5-3 · IND5-5 · IND5-8 📂 Phase 3 — Alien Invasion Mechanics

Ship hits, lives, and game over

Learning Intentions

  • I can detect when an alien hits my ship
  • I can use a GameStats class to track lives
  • I can end the game when lives run out

Success Criteria

  • My ship dies when an alien hits it
  • I have 3 lives; the game ends when all are gone
  • Aliens reaching the bottom also counts as a hit
5 min

Do Now — Error of the Day

Play your current game for 2 minutes. Right now there's no way to lose. Does that feel right?

10 min

I Do — Live-coded demo

Live-code ship-alien collision using pygame.sprite.spritecollideany(). Create a GameStats class holding ships_left. On ship hit: decrement lives, pause briefly, reset fleet + ship. When lives = 0, game over.

15 min

We Do — Guided build

Class implements _ship_hit() and _check_aliens_bottom(). Test: wait for fleet to reach ship. Watch ship 'die'. Test: let aliens reach the bottom — same behaviour.

35 min

You Do — Your turn

Tune: 3 lives the right number? Add a sleep(0.5) pause between deaths (import time). Extension: display remaining ships as icons in the corner; use pygame.font to show 'GAME OVER'.

Support

Use lives_starter.py with _ship_hit pre-written.

Core

Build per Matthes Ch 14.

Extension

Add a combo system — consecutive hits without getting hit multiplies points.

5 min

Exit Ticket — Coding Journal

Journal: losing a life in a game is important. Why? What does it do to the player's emotional experience?

📝 Journal submission

Submit your 2-3 sentence journal entry in Google Classroom before you leave. Every entry contributes to your Coding Journal (/5) in the final assessment.
