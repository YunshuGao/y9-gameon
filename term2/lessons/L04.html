L04 · Week 2 ⏱ 75 minutes 🎯 IND5-3 · IND5-7 📂 Phase 1 — Python Foundations

Lists — storing many things in order

Learning Intentions

  • I can create a list and index into it
  • I can use .append() and .remove() to modify a list
  • I can find the length of a list with len()

Success Criteria

  • My team_roster.py has a list of 5+ items
  • I've indexed to get the first and last items
  • I've added one item with .append()
5 min

Do Now — Error of the Day

Error of the Day: print(team[5]) where team = ["Alice", "Bob", "Cara"]. What error do you get? What does this tell you about how Python counts?

10 min

I Do — Live-coded demo

Live-code: 

heroes = ["Mario", "Link", "Samus"]
print(heroes[0])  # first item
print(heroes[-1])  # last item — clever trick!
heroes.append("Kratos")
print(len(heroes))
Emphasise: Python indexes start at ZERO. This will bite every student at least once.

15 min

We Do — Guided build

Build team_roster.py: a list of 5 fictional game characters. Print each one on a new line using a for-loop preview: 

for character in roster:
    print(character)
(We'll learn for-loops properly next lesson — for now, recognise the shape.)

35 min

You Do — Your turn

Build inventory.py: a list of at least 5 items in a game inventory. Print each on a new line. Then: (1) add one item with .append(); (2) remove one with .remove(); (3) sort alphabetically with .sort(). Print the final list.

Support

inventory_starter.py with list defined; fill in the 3 actions.

Core

Complete inventory.py with all 3 actions + sorted output.

Extension

Use .insert(1, item) to add at specific position. Try sorted(inventory, reverse=True).

5 min

Exit Ticket — Coding Journal

Journal: lists use [] and strings use "". In 1 sentence: what's one thing lists can do that strings can't?

📝 Journal submission

Submit your 2-3 sentence journal entry in Google Classroom before you leave. Every entry contributes to your Coding Journal (/5) in the final assessment.
