L08 · Week 4 ⏱ 75 minutes 🎯 IND5-3 · IND5-5 · IND5-7 📂 Phase 1 — Python Foundations

Functions — reusable code blocks

Learning Intentions

  • I can define a function with def
  • I can pass parameters to a function
  • I can return a value from a function

Success Criteria

  • My adventure v3 has at least 3 functions
  • Each room is now a function call
  • My code is shorter and clearer than v2
5 min

Do Now — Error of the Day

Look at your adventure_v2.py. Count how many times you typed 'print(' in the code. Now imagine you wanted to change every print to add a border around it. How many edits?

10 min

I Do — Live-coded demo

Live-code: wrap repeated code into a function. 

def greet(name):
    print(f"Hello, {name}!")
    print(f"Welcome to our game.")
greet("Ava")
greet("Ben")
Introduce: parameters, arguments, return values, docstrings.

15 min

We Do — Guided build

Refactor together — wrap each adventure room in a function: 

def forest_room(player):
    print("You enter a dark forest.")
    choice = input("Go left or right? ")
    if choice == "left": return "treasure_room"
    else: return "pit"
Main loop calls the current room function; the function returns which room to go to next.

35 min

You Do — Your turn

Refactor your adventure_v2 → adventure_v3.py. Every room is a function. A main game loop dispatches: if current_room == 'forest': current_room = forest_room(). Aim for 4+ room functions + 1 main game function.

Support

Refactor just 2 rooms into functions. Keep rest as before.

Core

Full refactor of 4+ rooms into functions.

Extension

Add a fight(enemy) function that takes an enemy dict and returns win/lose — reuse across rooms.

5 min

Exit Ticket — Coding Journal

Journal: why do functions make code easier to maintain? Give one example from your adventure.

📝 Journal submission

Submit your 2-3 sentence journal entry in Google Classroom before you leave. Every entry contributes to your Coding Journal (/5) in the final assessment.
