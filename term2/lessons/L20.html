Lesson 20: Sprint 5 — Bug Hunt + Final Polish

Learning Intentions

  • Hunt every bug.
  • Polish the feel.
  • Submission-ready.

Success Criteria

  • My game runs without crashes for 3 minutes of play.
  • I've fixed at least 3 bugs I found.
  • My code has comments + a file-header description.

📍 Do Now — Error of the Day

Play YOUR game for 3 minutes right now. Note every single bug or weird thing you see. Don't fix yet — just note.

👁 I Do — Teacher Demo

Demo the bug-hunt protocol: sort by severity (crash > visual > feel), fix top 3, retest.

🤝 We Do — Build Together

Swap bug lists with a partner. They note anything you missed. Merge lists.

🚀 You Do — Your Turn

Fix your top 3 bugs. Write the file header comment. ZIP your game folder. TEST the zip in a new folder. If it runs — you're ready for tomorrow's playtest.

  • Play your own game for 3 min straight — note every bug
  • Fix the top 3 bugs
  • Write header comment with name, date, 1-sentence description
  • Add inline comments explaining any tricky code
  • Zip your game folder for submission
