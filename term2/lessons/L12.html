Drawing and moving your ship

Learning Intentions

  • I can load a sprite image and blit it to the screen
  • I can move a sprite with arrow keys
  • I can keep a sprite inside the screen bounds

Success Criteria

  • My ship appears on screen
  • My ship moves left/right with arrow keys
  • My ship doesn't leave the screen edges
5 min

Do Now — Error of the Day

Download ship.bmp from the hub. Save it in the same folder as alien_invasion.py.

10 min

I Do — Live-coded demo

Live-code a Ship class: 

class Ship:
    def __init__(self, screen):
        self.screen = screen
        self.image = pygame.image.load('ship.bmp')
        self.rect = self.image.get_rect()
        self.screen_rect = screen.get_rect()
        self.rect.midbottom = self.screen_rect.midbottom
    
    def blitme(self):
        self.screen.blit(self.image, self.rect)
Emphasise the concept of rect — every sprite has one; it tracks position.

15 min

We Do — Guided build

Integrate Ship into main loop: create ship = Ship(screen), call ship.blitme() after screen.fill(). See the ship appear. Now add keyboard: check for event.type == pygame.KEYDOWN, event.key == pygame.K_RIGHT → ship.rect.x += 1.

35 min

You Do — Your turn

Upgrade: continuous movement. In Ship class, add self.moving_right = False. On KEYDOWN set True, on KEYUP set False. Add update() method that moves while flag is True. Add bounds check: if self.rect.right >= self.screen_rect.right: moving_right = False.

Support

Use ship_class_starter.py. Fill in the blitme and update methods.

Core

Extension
