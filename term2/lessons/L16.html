L16 · Week 8 ⏱ 75 minutes 🎯 IND5-2 · IND5-3 · IND5-7 📂 Phase 3 — Alien Invasion Mechanics

Alien movement — side to side + drop

Learning Intentions

  • I can move the whole fleet together
  • I can detect when the fleet hits a screen edge
  • I can make the fleet drop down and reverse direction

Success Criteria

  • My fleet moves to the right
  • It drops down when hitting the right edge
  • It reverses and moves left, repeats
5 min

Do Now — Error of the Day

Draw on paper: how should aliens move? Trace the path. Discuss with partner for 2 min.

10 min

I Do — Live-coded demo

Live-code movement: 

def _check_fleet_edges(self):
    for alien in self.aliens.sprites():
        if alien.check_edges():
            self._change_fleet_direction()
            break

def _change_fleet_direction(self):
    for alien in self.aliens.sprites():
        alien.rect.y += self.settings.fleet_drop_speed
    self.settings.fleet_direction *= -1
Explain the *= -1 trick — multiplying direction by -1 flips it.

15 min

We Do — Guided build

Add Alien.update() method that uses fleet_direction. Add Alien.check_edges() that returns True if at screen edge. Integrate into main loop before aliens are drawn.

35 min

You Do — Your turn

Tune the movement: pick alien_speed (start at 1) and fleet_drop_speed (start at 10). Playtest — does it feel too fast? Too slow? Extension: speed up when all aliens in a row are destroyed (difficulty scaling).

Support

Use fleet_movement_starter.py.

Core

Build per Matthes Ch 13.

Extension

Add diagonal drift — aliens wobble up and down as they cross the screen.

5 min

Exit Ticket — Coding Journal

Journal: which is harder to write — fleet creation or fleet movement? Why?
