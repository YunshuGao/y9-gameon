L10 · Week 5 ⏱ 75 minutes 🎯 IND5-2 · IND5-3 · IND5-5 · IND5-7 📂 Phase 1 — Python Foundations

Checkpoint 1 — Text Adventure v3 submission

Learning Intentions

  • I can combine everything from Phase 1 into a working program
  • I can debug my own code when something breaks
  • I can submit a clean, commented Python file

Success Criteria

  • My adventure_v3.py runs without errors
  • I have a Character class with hero + at least 1 enemy
  • I've submitted to Google Classroom
5 min

Do Now — Error of the Day

Open your adventure_v3.py. Read through the first 20 lines. Are all variables clearly named? Fix any that aren't.

10 min

I Do — Live-coded demo

Teacher demonstrates final polish: add a game banner at start, add a 'credits' screen at end, show how to add comments at the top of the file (author, date, description). Demonstrate the PEP 8 naming convention: snake_case for variables, PascalCase for classes.

15 min

We Do — Guided build

In pairs: swap laptops and playtest each other's adventures. Report 1 bug found. Write feedback in the journal: what was clever about their code? What confused you?

35 min

You Do — Your turn

Finish adventure_v3.py to submission quality: • Header comment with author + description • Character class in use • At least 4 rooms as functions • At least 2 possible endings • Meaningful variable names throughout • At least 3 # comments explaining WHY, not just what Then submit to Google Classroom as adventure_v3_[yourname].py

Support

Use the simplified spec: Character class + 2 rooms + 2 endings. Still submit.

Core

Full spec as above.

Extension

Add a save-game function using a text file (preview of Ch 10 file I/O).

🔖 Checkpoint

✅ Checkpoint 1: Python Foundations complete. Next week we pivot to Pygame.

Download adventure_v3_reference.py

5 min

Exit Ticket — Coding Journal

Journal — BIG ENTRY this week: what are you proud of? What was the hardest part? What will you remember about Phase 1?

📝 Journal submission

Submit your 2-3 sentence journal entry in Google Classroom before you leave. Every entry contributes to your Coding Journal (/5) in the final assessment.
